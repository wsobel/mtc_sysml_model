# DataItem Types for Interface Package

Each [`Interface`]({% link model/InterfaceInteractionModel/Interface.md %}) contains data items which are used to communicate information required to execute the <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span>. When these data items are read by another piece of equipment, that piece of equipment can then determine the actions that it may take based upon that data.

[`InterfaceState`]({% link model/InterfaceInteractionModel/DataItemTypesforInterface/InterfaceState.md %}) is a data item specifically defined for <span class="hoverterm">interfaces<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span>.  It defines the operational state of the <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span>.  This is an indicator identifying whether the <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span> is functioning or not. See *Section InterfaceState* for complete semantic details.

Some data items **MAY** be directly associated with the [`Interface`]({% link model/InterfaceInteractionModel/Interface.md %}) element and others will be organized by a `References` element. It is up to an implementer to determine which additional data items are required for a particular <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span>.

### Specific Data Items for the Interaction Model for Interface

A special set of data items have been defined to be used in conjunction with [`Interface`]({% link model/InterfaceInteractionModel/Interface.md %}). They provide information from a piece of equipment to <span class="hoverterm">request<span class="definition" data-term="request">communications method where a *client* transmits a message to an *agent*.  That message instructs the *agent* to respond with specific information.</span></span> a service to be performed by another associated piece of equipment; and for the associated piece of equipment to indicate its progress in performing its <span class="hoverterm">response<span class="definition" data-term="response">response *interface* which responds to a *request*.</span></span> to the <span class="hoverterm">request<span class="definition" data-term="request">communications method where a *client* transmits a message to an *agent*.  That message instructs the *agent* to respond with specific information.</span></span> for service.  .

Many of the data items describing the services associated with an <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span> are paired to describe two distinct actions – one to <span class="hoverterm">request<span class="definition" data-term="request">communications method where a *client* transmits a message to an *agent*.  That message instructs the *agent* to respond with specific information.</span></span> an action to be performed and a second to reverse the action or to return to an original state.  For example, a [`DoorInterface`]({% link model/InterfaceInteractionModel/InterfaceTypes/DoorInterface.md %}) will have two actions [`OpenDoor`]({% link model/InterfaceInteractionModel/DataItemTypesforInterface/OpenDoor.md %}) and [`CloseDoor`]({% link model/InterfaceInteractionModel/DataItemTypesforInterface/CloseDoor.md %}). An example of an implementation of this would be a robot that indicates to a machine that it would like to have a door opened so that the robot could extract a part from the machine and then asks the machine to close that door once the part has been removed. 

When these data items are used to describe a service associated with an <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span>, they **MUST** have one of the following two `subType` elements: `REQUEST` or `RESPONSE`.  These **MUST** be specified to define whether the piece of equipment is functioning as the <span class="hoverterm">requester<span class="definition" data-term="requester">entity that initiates a *request* for information in a communications exchange.</span></span> or <span class="hoverterm">responder<span class="definition" data-term="responder">entity that responds to a *request* for information in a communications exchange.</span></span> for the service to be performed.  The <span class="hoverterm">requester<span class="definition" data-term="requester">entity that initiates a *request* for information in a communications exchange.</span></span> **MUST** specify the `REQUEST` `subType` for the data item and the <span class="hoverterm">responder<span class="definition" data-term="responder">entity that responds to a *request* for information in a communications exchange.</span></span> **MUST** specify a corresponding `RESPONSE` `subType` for the data item to enable the coordination between the two pieces of equipment.   

These data items and their associated `subType` provide the basic structure for implementing the <span class="hoverterm">interaction model<span class="definition" data-term="interaction model">model that defines how information is exchanged across an *interface* to enable interactions between independent systems.</span></span> for an <span class="hoverterm">interface<span class="definition" data-term="interface">means by which communication is achieved between independent systems.</span></span> and are defined in the following sections.

That message instructs the *agent* to respond with specific information.</span></span> and <span class="hoverterm">response<span class="definition" data-term="response">response *interface* which responds to a *request*.</span></span> respectively.
